# 鲲穹会纪 iOS 功能遗漏盘点

检查时间：2026-06-29

本次只检查 Android 已有功能、后端已暴露接口、当前 iOS 源码骨架之间的差异。不包含当前 Windows 主机完全无法完成的事项，例如 Xcode 编译、iOS 模拟器、真机录音验证、签名、TestFlight、StoreKit 沙盒实付验证。

## 1. 当前 iOS 已完成到什么程度

当前 iOS 目录为 `apps/ios_app/HuiyiApp`，是 Swift + SwiftUI 源码骨架，尚无 Xcode 工程。已经覆盖的主线如下：

- 登录/注册/验证码登录/重置密码：已接 `/auth/sms/*` 和 `/auth/password/*` 的基础登录链路。
- Token 持久化：已用 Keychain 封装 `TokenStore`。
- 会议首页基础列表：已通过 `/sync/bootstrap` 拉取任务和日程，展示会议列表、近期日程、导入和录音入口。
- 会议详情基础浏览：已展示纪要、议题、决策、风险、待办、转写原文，并支持待办完成状态同步、删除任务、本地 Markdown 分享。
- 文件导入基础链路：已支持语言选择、UIDocumentPicker、复制到沙盒、创建本地待处理任务。
- 录音/实时 ASR 源码骨架：已做 ASR 连接成功后再开始录音和计时、快速取消不创建任务、结束后进入处理队列。
- 处理队列基础链路：已支持开始处理、继续处理、失败可见、失败删除、终止处理、排队展示、轮询任务状态。
- 知识库基础问答：已接 `/knowledge/ask`，默认云端检索，能展示回答和来源标题。
- 会员中心基础展示：已展示当前权益、套餐、额度、订单列表，不调用 Android 支付宝 WAP。
- 个人中心基础展示：已展示账号信息、环境信息、退出登录、声纹档案入口。
- 声纹档案基础列表：已接 `/voiceprints/profiles` 读取列表。
- StoreKit 2：只有服务骨架，未接后端 Apple 交易接口。

## 3. 接口层遗漏

当前 iOS `APIClient` 已封装登录、会员读取、订单列表/详情、任务处理、知识库问答、声纹列表、文件上传。对照 Android `HuixiaoApiClient.kt` 和服务端路由，仍缺这些可在 Windows 先补源码的接口：

- 账号接口：`/auth/phone/change/verify-current`、`/auth/phone/change`、`/auth/password/set`、`/auth/password/change`。
- 日程接口：`PUT /sync/schedules/{schedule_id}`、`DELETE /sync/schedules/{schedule_id}`、`DELETE /sync/all`。
- 任务接口：`PATCH /tasks/{task_id}`、`GET /tasks/{task_id}/export`、`GET /tasks/{task_id}/audio`、`POST /tasks/{task_id}/regenerate-minutes`、`POST /tasks/regenerate-local-minutes`。
- 声纹接口：`POST /voiceprints/profiles/from-task`、`POST /voiceprints/profiles/from-audio`、`PATCH /voiceprints/profiles/{profile_id}`、`DELETE /voiceprints/profiles/{profile_id}`。
- 订单接口：iOS 可保留 `GET /payments/orders/{order_id}`；不应接 Android `/payments/alipay/...` 购买链路。后续需要新增 `/payments/apple/...`。
- 数据接口：`DELETE /data/local` 是测试/本地清理能力，正式 iOS 是否开放要谨慎。

## 4. 业务功能遗漏

### 4.1 任务和处理队列

已做：失败任务可见、点击不自动重跑、处理中任务优先、App 重启后处理中恢复为可继续。

还缺：

- 处理完成后自动跳会议详情的体验。
- 处理任务完成/失败后从首页、导入页、待办页多入口一致刷新。
- 额度不足、账号冻结、需要购买时的前置拦截和清晰状态。
- 本地任务删除时同时清理沙盒音频文件。
- 远程删除失败时的本地/远程一致性提示。

### 4.2 会议详情和纪要编辑

已做：查看纪要、议题、决策、风险、待办、原文；待办勾选同步；删除会议。

还缺：

- 编辑会议标题、参会人、标签、隐私、知识库范围。
- 编辑纪要正文、议题、决策、风险、原文片段。
- 说话人重命名并同步到原文和待办负责人。
- 从待办/知识来源跳回对应原文片段。
- 原文片段复制、音频片段播放、音频下载缓存。
- 重新生成纪要，并保留/合并用户已编辑的待办状态。
- 服务端导出 Markdown/TXT，而不是只用本地 Markdown 拼接。

### 4.3 待办中心

当前 iOS 没有独立待办页。Android 已有完整 `TasksScreen`，iOS 需要补：

- 待办聚合列表。
- 我的待办、今日、逾期、待确认筛选。
- 待办搜索。
- 待办详情页/弹窗。
- 创建手动待办。
- 编辑负责人、截止时间、优先级、描述、状态。
- 开始、完成、取消/删除待办。
- 跳转来源会议和原文片段。

### 4.4 日程和提醒

当前 iOS 只展示服务端返回的 schedules，不具备业务闭环。需要补：

- 新建预约会议。
- 编辑预约会议。
- 删除预约会议。
- 冲突检测。
- 从预约开始录音，并把 scheduleId、scheduleNote 固定到任务。
- 开始后移除/完成对应预约。
- 本地通知提醒或系统日历集成策略。
- 提醒稍后、忽略、开始会议。

### 4.5 知识库

已做基础问答。还缺：

- 会议知识主题列表。
- 来源点击跳转会议详情和原文片段。
- 多轮上下文传递。
- 回答重试/取消。
- 查询范围切换：云端、本地、全部。
- 额度不足/账号冻结前置提示。
- 本地知识片段参与问答。

### 4.6 声纹

当前只是列表页。还缺：

- 从会议说话人保存声纹档案。
- 录制 15-30 秒样本建档。
- 导入样本音频建档。
- 改名。
- 启用/停用。
- 删除。
- 声纹授权同意确认。
- 与会议详情中的说话人重命名联动。

### 4.7 会员、支付和订单

已做展示，不做支付宝 WAP。还缺：

- iOS StoreKit 商品 ID 和服务端套餐映射。
- 购买套餐。
- 购买加量包。
- 恢复购买。
- Apple 交易上报、服务端验证、权益发放。
- Apple Server Notifications。
- 独立订单记录页和订单详情页。
- 账号冻结、支付未配置、已有会员续购提示等细节。

这一块不应为了快去接 Android 支付宝 WAP。正式 iOS 必须走 StoreKit 2。

### 4.8 个人中心和账号

当前只展示账号信息和退出。还缺：

- 修改昵称。
- 手机号换绑。
- 设置密码。
- 修改密码。
- 云同步开关、手动拉取、上传未同步。
- 未同步会议数量。
- 权限状态：麦克风、通知、文件访问相关说明。
- 用户协议、隐私政策弹窗。
- 清理本地数据/云端数据入口是否保留的产品决策。

### 4.9 本地缓存和云同步

当前 iOS 有任务队列 JSON、音频沙盒、Token，但还缺 Android 那种完整离线/同步层：

- 会议 bootstrap 结果本地缓存，弱网时展示上次数据。
- 会员、订单、声纹列表缓存。
- 云同步操作队列：上传、更新结果、删除会议、同步日程。
- 云端音频下载缓存。
- 本地知识索引缓存。
- 搜索历史。

如果 iOS 首版强依赖远程数据库，可以暂时不做完整离线，但至少要做弱网缓存和失败重试，否则体验会明显弱于 Android。

## 5. 工程和配置遗漏

这些不是业务页面，但上线前必须补：

- Xcode project 或 Swift Package/项目生成方案。
- Bundle ID、Display Name、版本号、图标、启动屏。
- `Info.plist` 隐私权限文案：麦克风、文件访问、网络、通知等。
- ATS/HTTPS 策略。当前测试地址是 HTTP，iOS 正式环境应尽量使用 HTTPS。
- Entitlements：StoreKit、推送/通知、后台音频等按实际范围配置。
- Debug/Staging/Production 环境切换。
- App Store 审核材料：隐私政策、用户协议、账号删除/数据删除说明、IAP 恢复购买说明。

## 6. 不是漏做，但当前不能闭环验证

以下必须等 Mac/Xcode/真机：

- Swift 编译错误和 SwiftUI 预览。
- Keychain 行为。
- UIDocumentPicker 沙盒权限。
- AVAudioSession、AVAudioEngine、WAV 写入。
- 实时 ASR WebSocket 帧格式和延迟。
- 后台、锁屏、中断、来电、耳机切换。
- StoreKit 2 沙盒购买和恢复购买。
- 签名、IPA、TestFlight。

## 7. 建议下一步顺序

不改后端和数据库的前提下，Windows 侧可以继续补这些源码：

1. 先补 iOS 路由和页面骨架：全部会议、待办中心、预约会议、搜索、独立订单页。
2. 补 APIClient 缺失接口：账号、日程、任务元数据、导出、声纹增删改。
3. 补会议详情编辑闭环：会议信息、待办编辑、说话人重命名、来源定位。
4. 补个人中心：手机号换绑、设置/修改密码、协议隐私、云同步状态。
5. 补弱网缓存和本地文件清理。
6. 等 Mac 后第一轮只做编译和主链路跑通，不立刻做 StoreKit。

结论：当前 iOS 已经不是空壳，但离 Android 同等功能还有明显差距。最关键的缺口不是录音/ASR骨架，而是待办中心、会议详情编辑、日程预约、账号中心、声纹管理和接口覆盖。StoreKit/Apple 后端属于后续独立阶段，不建议混入当前 Android 支付宝语义。
