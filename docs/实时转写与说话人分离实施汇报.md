# 实时转写与匿名说话人分离实施汇报

## 当前范围

前期只实现两个核心能力：

- 实时转写
- 匿名说话人分离

声纹识别、真实姓名自动匹配、声纹库管理暂不实现。

## 当前状态

已完成：

- Android 端使用 `AudioRecord` 采集 `PCM 16k mono` 音频流。
- Android 端同步保存完整会议录音为本地 WAV 文件。
- Android 端通过 WebSocket 向后端发送实时 PCM 音频帧。
- App 实时转写列表只展示服务端返回内容，不使用假数据。
- `api_server` 新增 `/api/v1/live/ws`，作为 App 唯一实时入口。
- `api_server` 代理实时音频流到 `asr_service`。
- `asr_service` 新增 `/api/v1/live/ws`，作为 FunASR/CAMPPlus 实时网关。
- `asr_service` 按 FunASR WebSocket 协议发送 `2pass` 启动参数和 PCM 音频。
- `asr_service` 标准化 FunASR 返回结果为 `transcript.partial` / `transcript.final`。
- FunASR runtime 已部署为独立 WebSocket 服务，监听 `ws://127.0.0.1:10095`。
- FunASR runtime 使用 `2pass` 实时/离线修正链路，CAMPPlus 负责说话人 embedding。
- 匿名说话人分离使用 FunASR/3D-Speaker 的 `SpectralCluster` 对会话内 embedding 聚类，输出 `说话人 1/2/3`。
- 词级时间戳已转换为每段转写的 `start_ms/end_ms/timestamp`，App 可定位到对应录音时间。
- 文件上传解析已切换到同一套 `FunASR 2pass + CAMPPlus`，不再使用旧 Dolphin 的 18 秒文本切段和简化说话人聚类。
- 声纹识别、真实姓名匹配、声纹库管理本阶段未启用。

运行脚本：

```powershell
services\start_local_services.ps1
```

## 数据流向

```text
Android App
  AudioRecord 采集 PCM 16k mono
  本地写入完整 WAV 文件
  WebSocket 发送 PCM 帧
        |
        v
api_server
  ws://127.0.0.1:8080/api/v1/live/ws
  只做统一入口和转发
        |
        v
asr_service
  ws://127.0.0.1:8081/api/v1/live/ws
  转换为 FunASR WebSocket 协议
        |
        v
FunASR 2pass + CAMPPlus Runtime
  ws://127.0.0.1:10095
  VAD / 实时转写 / 离线修正 / CAMPPlus embedding / 匿名说话人聚类
        |
        v
asr_service 标准化结果
        |
        v
api_server 转发结果
        |
        v
Android App 实时转写列表
```

## API 配置

Android：

```text
HUIXIAO_API_BASE_URL=http://127.0.0.1:8080/api/v1
HUIXIAO_LIVE_WS_URL=ws://127.0.0.1:8080/api/v1/live/ws
```

`api_server`：

```text
HUIXIAO_ASR_SERVICE_URL=http://127.0.0.1:8081
HUIXIAO_ASR_LIVE_WS_URL=ws://127.0.0.1:8081/api/v1/live/ws
```

`asr_service`：

```text
HUIXIAO_FUNASR_RUNTIME_WS_URL=ws://127.0.0.1:10095
HUIXIAO_LIVE_ASR_MODE=2pass
HUIXIAO_LIVE_CHUNK_SIZE=5,10,5
HUIXIAO_LIVE_CHUNK_INTERVAL=10
HUIXIAO_LIVE_ENCODER_LOOK_BACK=4
HUIXIAO_LIVE_DECODER_LOOK_BACK=1
HUIXIAO_LIVE_REQUIRE_SPEAKER=true
```

## WebSocket 事件

App 发送：

```json
{"type":"audio.start","format":"pcm_s16le","sample_rate":16000,"channels":1,"speaker_separation":true}
```

随后持续发送二进制 PCM 音频帧。

App 停止录音：

```json
{"type":"audio.end"}
```

服务端返回：

```json
{
  "type": "transcript.final",
  "session_id": "live-xxx",
  "segments": [
    {
      "speaker": "说话人 1",
      "text": "我们先复盘上周活动效果",
      "timestamp": "00:01",
      "start_ms": 1200,
      "end_ms": 5600
    }
  ],
  "raw_has_speaker": true
}
```

错误返回：

```json
{"type":"error","message":"FunASR/CAMPPlus 实时服务不可用：..."}
```

## 数据存储

- 原始录音：默认保存在 App 本地私有目录。
- 实时转写中间数据：当前只在 App 内存显示，不落库。
- 会议结束后的完整处理结果：继续走现有上传处理链路，写入 `HUIXIAO_DATABASE_URL` 指向的共享远程数据库。
- 声纹 embedding：本阶段不创建、不保存。
- OSS：本阶段不用于实时转写和说话人分离。
