# 鲲穹会纪 iOS Mac Verification Checklist

This iOS source tree was prepared on Windows. The items below must be verified on macOS with Xcode before claiming the iOS app is runnable.

## Source Prepared On Windows

- SwiftUI app entry, routing, and main tab structure.
- XcodeGen `project.yml`, Info.plist, minimal entitlements, AppIcon, and AccentColor assets.
- Login, SMS login, registration, and password reset screens.
- URLSession API client for auth, sync, tasks, results, knowledge, membership, orders, live sessions, uploads, and voiceprints.
- Keychain token storage boundary.
- Codable models aligned with existing FastAPI schemas.
- Meeting list and cloud bootstrap.
- Meeting detail with summary, topics, decisions, risks, todos, searchable transcript, source inspection, transcript correction, speaker editing, meeting info editing, todo completion/start/edit/delete/manual create, delete, and Markdown share.
- Knowledge Q&A through `/knowledge/ask`, including local/cloud/all scope switch, recommended questions, topic cards, context follow-up, source jump, cancel, and retry edit.
- Membership display, plans, add-ons, quotas, order list, and iOS purchase-disabled state.
- File import flow with language selection, UIDocumentPicker wrapper, sandbox copy, local queue creation, upload-before-processing bridge.
- Processing page with explicit start/retry, polling, cancel, failed-task visibility, and delete-failed behavior.
- Recording page state machine: ASR prepare first, record only after ASR ready, cancel before start without creating a task, pause/resume, schedule context, consent sheet, enqueue after stop.
- Realtime ASR client wrapper for direct Aliyun WebSocket credentials from `/live/session`.
- Recording service skeleton using AVAudioSession, AVAudioEngine, PCM conversion, and WAV writing.
- StoreKit 2 service skeleton for future Apple purchases and restore.
- Voiceprint profile list, rename, active toggle, delete, import sample, and record sample through the existing voiceprint APIs.
- Schedule list/create/edit/delete, local notification scheduling, reminder snooze/dismiss/start recording flow.

## Must Verify On Mac/Xcode

- Xcode project generation from `project.yml` and target settings.
- Bundle ID, display name, entitlements, and Info.plist privacy strings.
- Swift compilation and strict concurrency warnings.
- SwiftUI layout on small devices.
- Keychain save/load behavior.
- Login and cloud API calls against staging.
- UIDocumentPicker security-scoped URL access and sandbox copy.
- File upload multipart request compatibility.
- Processing polling and cancellation against staging.
- AVAudioSession microphone permission flow.
- AVAudioEngine input format, PCM conversion, audio level, and WAV header correctness.
- Realtime ASR WebSocket connection, session update, final/partial transcript parsing, and finish behavior.
- Fast tap stop before ASR ready: must not create a task.
- Recording stop after ASR ready: must enqueue a task and enter processing.
- Background/lock-screen recording policy and App Review compliance.
- StoreKit 2 products, purchase, restore, transaction verification, sandbox testing.
- TestFlight signing, archive, upload, and real-device smoke test.

## Not Implemented Yet

- Committed `.xcodeproj` generated by Xcode/XcodeGen on macOS.
- Apple `/payments/apple/...` backend APIs and Apple transaction database table.
- StoreKit product ID mapping from server.
- Server-side Apple transaction verification.
- Calendar integration with the system Calendar app.
- Push notifications. Local schedule reminders are implemented and still need device verification.
